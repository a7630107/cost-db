## 4. 技术架构与选型

### 4.1 总体架构（在线表格驱动）

```
[owner 浏览器] ──改数据──> [飞书多维表格 / 腾讯文档]  (数据源)
                                          │
                                   （定时/按需触发）
                                          ▼
                                  [轻量云函数]  (桥接：表格→JSON，解决鉴权/CORS/缓存)
                                          │
                                          ▼
                              [静态前端网站]  (现有原型改造，fetch 云端 JSON)
                                          │
                                          ▼
                                  [云静态托管+CDN]  (自带 HTTPS，一直在线)
```

### 4.2 关键决策点（写进需求书，避免外包跑偏）

| 决策 | 选型 | 理由 |
|---|---|---|
| 数据源 | **飞书多维表格**（或腾讯文档） | owner 非技术，表格可可视化维护，零数据库运维 |
| 桥接层 | **轻量云函数**（腾讯云 SCF / 阿里云 FC） | 解决三件事：① 表格 API 的鉴权 token 不暴露在前端 ② 绕过浏览器跨域(CORS) ③ 限流+缓存提速 |
| 前端 | 现有静态原型改造 | 只把「读本地 data.js」改成「fetch 云端 JSON」，改动极小 |
| 托管 | 对象存储静态托管 + CDN（腾讯云 COS / 阿里云 OSS） | 自带 HTTPS、一直在线、成本极低 |
| 域名 | 自备域名 + 免费 SSL 证书（Let's Encrypt） | 专业形象，证书免费 |

> **为什么不能直接前端 fetch 在线表格？**
> 在线表格的读取接口需要鉴权 token，若写在前端等于把密钥公开；且浏览器直连通常有跨域限制。所以必须加一个**极轻云函数**做「代理 + 缓存」，这是本方案唯一的后端成分，但开发量很小。

## 5. 数据更新操作流程（给 owner / 运营）

1. 打开飞书多维表格（已建好模板，字段见第 7 节）。
2. 在表格里**增 / 改 / 删**造价数据（如新增一条材料价）。
3. 触发云函数刷新（可手动点一下，或设定时自动每 N 分钟刷）。
4. 刷新后网站**自动展示新数据**，无需改代码、无需发包。

## 9. 风险与注意

| 风险 | 说明 | 应对 |
|---|---|---|
| 表格 API 限流 | 在线表格读取有频率限制 | 云函数做缓存，前端读缓存 JSON 而非直连表格 |
