## 三、怎么让别人也能访问（部署）

**方式A：发文件（最简单）**
把整个文件夹发给同事，对方双击 `index.html` 即可。数据自带在 `data.json` 里。

**方式B：放到网上（推荐，支持"刷新数据"按钮）**
用 GitHub Pages（免费）：
1. 在 GitHub 新建一个仓库（例如 `cost-db`）。
2. 把 `造价数据库原型` 文件夹里的**全部内容**（含 `.github`、`feishu.config.json`、各网站文件）推送到 `main` 分支。
3. 仓库 **Settings → Pages → Build and deployment**：Source 选 `Deploy from a branch`；Branch 选 `main`、目录选 `/ (root)` → Save。
4. 等一两分钟，访问 `https://<你的用户名>.github.io/<仓库名>/` 即可。
5. 点网页上的「刷新数据」会重新拉取 `data.json`。

## 四、全自动更新（飞书改 → 网站自动变）

由仓库里的 `.github/workflows/sync.yml` 完成：每 6 小时自动跑一次 `feishu_sync.js`，把飞书数据重写进 `data.json` 并提交，GitHub Pages 随即更新。

**只需配置一次（都在 GitHub 网页上点）：**

1. 进入仓库 **Settings → Secrets and variables → Actions → New repository secret**，添加 3 条（名称必须完全一致）：

   | Secret 名称 | 值 | 说明 |
   |------|------|------|
   | `FEISHU_APP_ID` | `cli_aad8588c14789bc1` | 飞书应用 ID |
   | `FEISHU_APP_SECRET` | `（在此粘贴你的飞书 app_secret）` | 飞书应用密钥 **只放 Secrets，绝不写进任何文件** |
   | `FEISHU_APP_TOKEN` | `XFiabvpAHat3X6sM6xIcDxGqnMg` | 飞书多维表格应用 token |

2. 之后无需任何操作：
   - 在飞书改表格 → 最多 6 小时后网站自动更新；
   - 想立刻更新：进仓库 **Actions** 页 → 选中 `飞书同步到造价数据库` → **Run workflow** 手动触发。

> 工作流提交的 `data.json` 带 `[skip ci]` 标记，不会自己反复触发，安全无循环。

## 六、文件清单

| 文件 | 作用 |
|------|------|
| `index.html` | 网页入口 |
| `style.css` | 样式 |
| `app.js` | 交互逻辑（fetch data.json + 搜索/导出/刷新） |

| `feishu.config.json` | 配置（app_token + 各表 ID；不含密钥） |
| `.github/workflows/sync.yml` | GitHub Actions 自动同步工作流（每 6 小时） |
| `运行与更新说明.md` | 本文件 |
